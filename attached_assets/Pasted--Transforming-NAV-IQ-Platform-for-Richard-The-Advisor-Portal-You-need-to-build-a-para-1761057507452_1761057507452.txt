## **1. Create Dedicated "Advisor Portal" (New User Type)**

### **Current Problem:**
Your platform assumes the **GP is the user**. But Richard is the **intermediary** - he's managing multiple GPs and needs to submit deals on their behalf while maintaining confidentiality and tracking competitive processes.

### **New Architecture:**

```
┌─────────────────────────────────────────┐
│         User Type Routing                │
├─────────────────────────────────────────┤
│                                          │
│  /gp/onboarding  → Current GP Flow       │
│  (Keep existing)                         │
│                                          │
│  /advisor/dashboard → NEW Richard Flow  │
│  (Build this)                            │
└─────────────────────────────────────────┘
```

### **Implementation:**

**Add to database schema:**
```typescript
advisors: {
  id: string (UUID)
  firmName: string  // "FFP Advisory"
  advisorName: string  // "Richard Wheelahan"
  email: string
  phone: string
  linkedInUrl: string | null
  commission_rate: number  // 0.50-0.75 (50-75 bps)
  status: 'active' | 'inactive' | 'pending'
  deals_submitted: number
  deals_won: number
  total_volume: number  // $ closed through platform

advisor_deals: {
  id: string (UUID)
  advisorId: string (FK)

  status: 'draft' | 'submitted' | 'competing' | 'won' | 'lost'
  submission_date: Date
  lenders_invited: string[]  // ["NAV IQ", "17Capital", "Hayfin"]

}
```

